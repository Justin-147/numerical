## 基于 R 值的预报效能评估（单测项）

本程序用于对**单个测项**的已提取特征序列进行预报效能评估：读取“待评估数据文件（时间+数值）”、地震目录、多边形规则与干扰信息，在「阈值 × 预报期」网格上计算 R 值，输出 R_{max}、R_0、报准/漏报统计及图件。

### 1. 程序与依赖

- **主程序**：`R_value_compute.py`
- **依赖安装**（建议 Python 3.9+）：

```bash
pip install -r requirements.txt
```

- **可选**：若需在空间分布图中使用 **cartopy** 底图，在程序中设置 `USE_CARTOPY_MAP = True` 并安装：`pip install cartopy`。

### 2. 输入文件


| 文件           | 说明                                                                                              |
| ------------ | ----------------------------------------------------------------------------------------------- |
| **待评估数据文件**  | 文本，两列：第 1 列时间码，第 2 列数值。时间支持：日值 `yyyymmdd`、整点值 `yyyymmddhh`、分钟值 `yyyymmddhhmm`。可有/无表头（表头可中文/英文）。 |
| **多边形规则**    | 文本，首行为标题（可含中文），其后两列：`经度`、`纬度`，每行一个顶点，首尾闭合。                                                      |
| **地震目录**     | EQT 定长格式，每行包含年月日时分秒、经纬度、震级、深度等。                                                                 |
| **干扰信息**（可选） | 文本文件。首行为标题，前两列：干扰开始时间、结束时间（时间码格式同待评估数据文件）。不存在或为空时按无干扰处理。                                        |


> 注：待评估数据文件 / 多边形规则 / 地震目录若不存在或内容为空，程序会报错并退出；干扰文件可为空或不存在。

### 3. 报准的判定标准

- **预报窗定义**：对每一段异常区间 [t_{st}, t_{ed}]（开始时间、最后时间，单位：天），其对应的预报窗为：
  - 若 **段长 > 预报期**（即 t_{ed} - t_{st} > 预报期天数）：预报窗 = [t_{st}, t_{ed}]
  - 若 **段长 ≤ 预报期**：预报窗 = [t_{st}, t_{st} + \text{预报期}]（即从异常开始起算、长度为预报期）
- **报准判定**：将地震发生时间取整到**天**（向下取整 \lfloor t_{eq} \rfloor）。若该整数日落在**任意一段**异常所对应的预报窗内（区间闭区间，即 t_{st} \leq \lfloor t_{eq} \rfloor \leq t_{ed}），则该地震记为**报准**；否则为漏报。
- **小结**：一个地震只要被任意一段异常的预报窗覆盖（按“天”对齐），即计为一次报准；多段异常可重复报准同一地震，但统计报准数时每震只计一次。
- R 值计算：报准率 - 时间占用率
- 报准率：报准地震数/地震总数
- 时间占用率：异常段长度之和/总时间跨度

### 4. 配置与运行

在 `R_value_compute.py` 末尾 `if __name__ == "__main__":` 中修改参数后运行：

```bash
python R_value_compute.py
```

关键可调参数（在 `test_example == 1/2` 分支内）：

- **DATA_FILE / POLYGON_FILE / EQ_CATALOG_FILE / INTERFERENCE_FILE**：输入路径
- **STATION_LON / STATION_LAT**：台站经纬度
- **DATA_HAS_HEADER / DATA_DELIMITER**：数据文件是否含表头、分隔符（默认 `\t`）
- **MAG_MIN / MAG_MAX**：地震目录筛选震级范围
- **ALM_DAY_START / ALM_DAY_END / ALM_DAY_STEP**：预报期（天）序列
- **THRESHOLD_MODE**：
  - `1`：阈值按“数据的若干倍 STD”扫描（`RATE_MIN / RATE_MAX / RATE_STEP`）
  - `2`：阈值按“给定阈值范围（数据单位）”扫描（`THRESH_START / THRESH_END / THRESH_STEP`）
- **USE_CARTOPY_MAP**：空间分布图是否启用 cartopy 底图（默认 False）

### 5. 流程概览

1. 读取待评估数据文件（两列：时间、数值），时间统一转换为内部儒略日。
2. 读取多边形规则；从地震目录按时间范围、震级范围与多边形范围筛选震例；读取干扰时段（若有）。
3. 在 (阈值 × 预报期) 网格上计算 R 矩阵，取 R_{max} 及对应阈值、预报期；计算 R_0 与报准/漏报。
4. 输出结果文件 `<数据文件名>_result.txt` 与图件 `<数据文件名>_result.png`。

### 6. 输出文件

- `**<数据文件名>_result.txt`**：预报效能评估结果（R_{max}、R_0、最优阈值与预报期、异常起止、报准/漏报及震例信息等）。
- `**<数据文件名>_result.png`**：三幅子图（空间分布、R–TT 曲面、时序阈值/预报窗/震例）。

所有输出文件与待评估数据文件同目录。
